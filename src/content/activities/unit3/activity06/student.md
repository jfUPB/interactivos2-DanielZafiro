#### ¬øC√≥mo se comunican los clientes con el servidor?

Los clientes se comunican con el servidor mediante **WebSockets**, usando la librer√≠a `socket.io`. Esta tecnolog√≠a permite establecer una conexi√≥n persistente y bidireccional entre cada cliente (mobile o desktop) y el servidor.

#### ¬øC√≥mo se comunican los clientes entre s√≠?

Los clientes no se comunican directamente entre ellos. Toda la comunicaci√≥n pasa por el servidor. El servidor act√∫a como intermediario: cuando un cliente env√≠a un mensaje, el servidor lo recibe y luego lo retransmite al otro cliente mediante `socket.broadcast.emit()` o `socket.to(...).emit()`.

#### Tipo de mensajes, datos y eventos

- ¬øQu√© tipo de mensajes se env√≠an?
  - Se env√≠an dos tipos principales de mensajes:
    - `image`: mensaje que contiene los datos binarios de la imagen capturada o seleccionada por el cliente mobile.
    - `feedback`: mensaje que contiene la reacci√≥n (like o dislike) enviada por el cliente desktop al recibir la imagen.

- ¬øQu√© tipo de datos se env√≠an?
  - **Datos binarios**: el contenido de la imagen se env√≠a como un `ArrayBuffer`, lo cual permite transferir archivos eficientemente.
  - **Datos JSON**: el feedback del cliente desktop se env√≠a como un objeto JSON simple que incluye la propiedad `type` con el valor `'like'` o `'dislike'`.

-  ¬øQu√© tipo de eventos se generan?
    -  `connect` y `disconnect`: eventos autom√°ticos de conexi√≥n/desconexi√≥n de cada cliente.
    -  `image`: evento personalizado generado por el cliente mobile para enviar una imagen al servidor.
    -  `feedback`: evento personalizado generado por el cliente desktop para enviar una reacci√≥n.
    -  `feedback` (tambi√©n): evento que el servidor reenv√≠a al cliente mobile con la reacci√≥n recibida.

#### Flujo de datos

- ¬øC√≥mo es el flujo de datos entre los clientes y el servidor?

1. El cliente mobile selecciona o captura una imagen.
2. Env√≠a la imagen al servidor mediante el evento `image`.
3. El servidor recibe el evento `image` y retransmite los datos al cliente desktop.
4. El cliente desktop muestra la imagen y espera una reacci√≥n del usuario.
5. Al seleccionar like/dislike, el cliente desktop emite un evento `feedback`.
6. El servidor recibe el feedback y lo reenv√≠a al cliente mobile.

- ¬øC√≥mo es el flujo de datos entre los clientes?

Aunque no existe comunicaci√≥n directa entre clientes, el flujo entre ellos es el siguiente:

1. **Cliente Mobile ‚Üí Servidor ‚Üí Cliente Desktop**: flujo de datos de la imagen.
2. **Cliente Desktop ‚Üí Servidor ‚Üí Cliente Mobile**: flujo del feedback.

Ambos clientes dependen del servidor como puente para intercambiar mensajes y datos.
